# Documentation Index

All the docs live here, organized by what they help you do. Most people start with the root README, but depending on what you're working on, you might jump straight to architecture if you want to understand the decision logic, or deployment if you want to get something running.

## What's Where
## System Documentation

This folder contains the complete, detailed documentation for the ESP32 Smart Plant Care system. Read these files in order to fully understand, deploy, and maintain the project.

- `01_architecture.md`: Complete system design, data flow, and technical details
- `02_hardware.md`: Hardware requirements and wiring overview
- `03_firmware.md`: Software structure, build commands, and API design
- `04_calibration.md`: Sensor calibration procedure and thresholds
- `05_deployment.md`: Step-by-step deployment procedures
- `06_validation.md`: Test cases, scenarios, and expected output
- `07_troubleshooting.md`: Real-world failures and resolution guides

### Quick Links
- **I want to understand how it works:** Start with `01_architecture.md`
- **I want to build the hardware:** Start with `02_hardware.md` and check `hardware/wiring/build-guide.md`, then come back to `07_troubleshooting.md` if something doesn't work.
- **I want to test it without building:** Check `simulation/wokwi/README.md` and run the scenarios in `06_validation.md`.
